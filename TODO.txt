AHRS / notch filtering notes:
  - ArduPilot applies harmonic notch filters in software (biquad on CPU).
  - FFT is only used to estimate notch center frequencies; the filter itself is still biquad.
  - Invensense IMU driver configures AAF/LPF; internal IMU notch is left at default.
  - ArduPilot uses CMSIS-DSP mainly for FFT (GyroFFT), not for notch filters.
  - CMSIS-DSP in ArduPilot is shipped as prebuilt DSP libs + headers.
  - Minican plan (recent Invensense 6-axis IMU):
    * Use gyro notch from bidir DShot RPM (4 motors + harmonics).
    * Update notch coefficients at low rate (e.g., 50-200 Hz).
    * Optional FFT-based tracking for non-motor resonances if CPU budget allows.
    * Keep IMU AAF/LPF enabled as the first stage.

CODE MODERNIZATION (C++26 / GCC 15.2):

  ° DeviceStatus Monadic Flow:
    + add DeviceStatus::and_then(F&& f) to allow chaining role initialization.
    + flatten UAVCanSlave.cpp::start() by replacing the pyramid of 'if(ok)' with a monadic pipeline.

  ° Flash Optimization (Remove snprintf):
    + replace snprintf calls with std::to_chars (constexpr in C++26) for numeric conversions.
    + objective: remove the expensive printf/formatting engine from the binary (gain ~5-10KB).

  ° String Handling (Modern std::string_view):
    + replace manual size checks and strncmp with C++23 methods: starts_with(), ends_with(), contains().
    + impact: cleaner code, better compiler optimization, zero RAM overhead.

  ° Template Simplification (Structural Types):
    + replace FixedString wrapper in addRole<...>() with C++20 StringLiteral structural type.
    + allows passing "ROLE.name" directly as a template parameter.

  ° Variant/Visitor Modernization:
    + use a variadic 'overload' lambda helper for variant/StoredValue visiting in ttyConsole.cpp.
    + removes the need for separate visitor structs (OverloadDyn).

  ° DSDL Static Assignment:
    + use consteval helpers for assigning literal strings to DSDL fields (text.len + text.data).
    + ensures length validation at compile-time and optimized register-to-memory transfers.

  ° Chrono Integration (Glue for ChibiOS):
    + implement a chibios_clock for std::chrono integration.
    + allows using 300ms, 1s literals safely while mapping to chibios ticks (sysinterval_t).
